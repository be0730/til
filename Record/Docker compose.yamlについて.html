⭐️compose.yamlの作成 services:　① web:　② image: httpd:2.4　③ ports: - "8080:80"　④ networks: ⑤ volumes: volweb01:　⑥ ①「services」 コンテナの定義を書くところ Docker Composeファイルではサービス＝コンテナと考えて問題ない。 複数のコンテナを作成する作成する際は、ここの配下にコンテナ定義を複数記載する ②「コンテナ名」 コンテナの名前を記載する ③「イメージ」 使用するイメージを記載する。ここでは「httpd」のバージョン2.4を指定 ④「ポート番号」 Dockerホストとコンテナのポート番号を紐付け 「ホスト側ポート:コンテナ側ポート」の形式で記載 ⑤「ネットワーク」 コンテナ間の通信に使用するネットワークを定義するところ ここではデフォルトネットワークを使用するため、空欄のままにしている ⑥「ボリューム」 コンテナで使用するボリュームを定義するところ(Dockerが管理する記憶領域にデータを永続化する仕組み) コンテナ削除後も、コンテナ作成時にそのボリュームをマウントすることで、データを引き継ぐことができる。 ボリュームとバインドマウントの比較 |機能|データの保存場所|作成方法| |ボリューム|Dockerが管理する記憶領域|「volumes」を記述。「services」下の「volumes」にボリュームの名前を書く| |バインドマウント|Dockerホストのフォルダ|「services」下の「volumes」に、ホストOSとコンテナのフォルダーの紐付けを記載する| ⭐️ボリュームとバインドマウントの例 （ボリューム） services: コンテナ名: image: イメージ名 volumes: - 「volumes」に定義したボリューム名:コンテナ内のフォルダパス ←バインドマウントの例 volumes: ボリューム名 （バインドマウント） services: コンテナ名: image: イメージ名 volumes: - ホストOSのフォルダパス:コンテナ内のフォルダパス ・コンテナに環境変数を設定するには environmentという項目に記述する。 depemds_onという項目を使用すると、他のコンテナの起動を待ってから起動するようにできる。 例） services: web: image: httpd:2.4 ports: - "8080:80" environment: - ENV_VAR_NAME=value depends_on: - db db: image: mysql:5.7 environment: - MYSQL_ROOT_PASSWORD=rootpassword - MYSQL_DATABASE=mydatabase ⭐️DockerFileに記述できる命令 FROM ベースイメージを指定 RUN イメージ作成時に実行するコマンドを指定 COPY ホストのファイルをイメージ内にコピー ADD ホストのファイルをイメージ内にコピー（COPYと似ているが、URLからの取得や圧縮ファイルの展開も可能） CMD コンテナ起動時に実行するコマンドを指定（RUNと異なり、コンテナ起動時に実行される） EXPOSE 公開するポートを指定。あくまでどのポートを公開するかを示すだけで、実際にポートを開くわけではない 実際のポートを開くには、docker runコマンドの-pオプションやcomposeファイルのportsを使用する ENTRYPOINT コンテナ起動時に実行するコマンドを指定（CMDと似ているが、引数の追加が可能） WORKDIR 作業ディレクトリを指定
